  <action>
    Create `backend/app/services/chat.py` following the research recommendation exactly. The module must contain:

    1. **Pydantic models** for structured output:
       - `TradeAction(BaseModel)`: ticker (str), side (str), quantity (float)
       - `WatchlistChange(BaseModel)`: ticker (str), action (str)
       - `ChatResponse(BaseModel)`: message (str), trades (list[TradeAction] default_factory=list), watchlist_changes (list[WatchlistChange] default_factory=list)

    2. **SYSTEM_PROMPT** constant — FinAlly AI trading assistant persona. Include instructions to respond with valid JSON matching the schema. Be concise and data-driven. Mention trade/watchlist capabilities.

    3. **format_portfolio_context(portfolio, watchlist)** function — formats cash, total value, positions with P&L, and watchlist tickers with prices into a readable string block.

    4. **mock_response(user_message)** function — keyword-based deterministic mock:
       - "buy" in message -> returns TradeAction(ticker="AAPL", side="buy", quantity=10)
       - "sell" in message -> returns TradeAction(ticker="AAPL", side="sell", quantity=10)
       - "add" and "watchlist" in message -> returns WatchlistChange(ticker="PYPL", action="add")
       - "remove" and "watchlist" in message -> returns WatchlistChange(ticker="PYPL", action="remove")
       - default -> generic portfolio help message with no actions

    5. **async handle_chat_message(user_message, price_cache, data_source)** function:
       - Load portfolio via `get_portfolio(price_cache)`
       - Load watchlist via `get_watchlist(price_cache)`
       - Query last 10 messages: `SELECT role, content FROM chat_messages WHERE user_id = ? ORDER BY created_at DESC LIMIT 10` then reverse
       - Build messages list: system prompt + context block, history, user message
       - If `os.environ.get("LLM_MOCK", "").lower() == "true"`: call mock_response
       - Else: call `acompletion(model="gemini/gemini-3.1-flash-lite-preview", messages=messages, response_format=ChatResponse, api_key=os.environ.get("GOOGLE_API_KEY", ""))`, parse with `ChatResponse.model_validate_json(response.choices[0].message.content)`. Wrap in try/except for ValidationError — on failure, return error message to user.
       - Auto-execute trades: iterate parsed.trades, call `execute_trade(price_cache, trade.ticker.upper(), trade.side, trade.quantity)`, collect (result, error)
       - Auto-execute watchlist changes: iterate parsed.watchlist_changes, call add_ticker or remove_ticker in try/except
       - Append all errors to message
       - Store user message and assistant message in chat_messages (with actions JSON for assistant)
       - Return `{"message": final_message, "actions": {"trades": [...], "watchlist_changes": [...]}}`

    Use `from __future__ import annotations` at top. Use `logging.getLogger(__name__)` for logger. Follow all conventions from CONVENTIONS.md.
