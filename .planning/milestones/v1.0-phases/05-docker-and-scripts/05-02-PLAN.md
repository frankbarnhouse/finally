<task type="auto">
  <name>Task 1: Create macOS/Linux start and stop scripts</name>
  <files>scripts/start_mac.sh, scripts/stop_mac.sh</files>
  <read_first>planning/PLAN.md</read_first>
  <action>
Create `scripts/` directory if it does not exist.

1. Create `scripts/start_mac.sh` (bash script):
- Shebang: `#!/usr/bin/env bash`
- `set -e`
- Define constants: `IMAGE_NAME="finally"`, `CONTAINER_NAME="finally-app"`, `VOLUME_NAME="finally-data"`, `PORT=8000`
- Resolve the project root directory (parent of scripts/): `SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"` then `PROJECT_DIR="$(dirname "$SCRIPT_DIR")"`
- Check if Docker is running; if not, print error and exit
- Accept `--build` flag to force rebuild. Also auto-build if image doesn't exist (`docker image inspect` returns non-zero).
- Build: `docker build -t "$IMAGE_NAME" "$PROJECT_DIR"`
- Check if container already running; if so, print message and skip (idempotent)
- Remove any stopped container with same name: `docker rm "$CONTAINER_NAME" 2>/dev/null || true`
- Run: `docker run -d --name "$CONTAINER_NAME" -v "$VOLUME_NAME":/app/db -p "$PORT":8000 --env-file "$PROJECT_DIR/.env" "$IMAGE_NAME"`
- Print: "FinAlly is running at http://localhost:$PORT"
- If `--open` flag passed (or as second arg), open browser: `open "http://localhost:$PORT"` on macOS, `xdg-open` on Linux

2. Create `scripts/stop_mac.sh` (bash script):
- Shebang: `#!/usr/bin/env bash`
- `set -e`
- Define same constants: `CONTAINER_NAME="finally-app"`
- Stop the container: `docker stop "$CONTAINER_NAME" 2>/dev/null || true`
- Remove the container: `docker rm "$CONTAINER_NAME" 2>/dev/null || true`
- Print: "FinAlly stopped. Data preserved in Docker volume."
- Do NOT remove the volume (data persists)
- Script is idempotent (no error if container not running)

3. Make both executable: `chmod +x scripts/start_mac.sh scripts/stop_mac.sh`

  <action>
Port the bash scripts to PowerShell, maintaining the same logic and constants.

1. Create `scripts/start_windows.ps1`:
- No shebang (PowerShell)
- `$ErrorActionPreference = "Stop"`
- Define same constants: `$ImageName = "finally"`, `$ContainerName = "finally-app"`, `$VolumeName = "finally-data"`, `$Port = 8000`
- Resolve project root: `$ScriptDir = Split-Path -Parent $MyInvocation.MyCommand.Path` then `$ProjectDir = Split-Path -Parent $ScriptDir`
- Check Docker is running: `docker info` in try/catch
- Accept `-Build` switch parameter to force rebuild. Auto-build if image doesn't exist.
- Build: `docker build -t $ImageName $ProjectDir`
- Check if container already running; if so, print message and exit
- Remove stopped container: `docker rm $ContainerName 2>$null` (ignore errors)
- Run: `docker run -d --name $ContainerName -v "${VolumeName}:/app/db" -p "${Port}:8000" --env-file "$ProjectDir\.env" $ImageName`
- Print: "FinAlly is running at http://localhost:$Port"
- If `-Open` switch passed, open browser: `Start-Process "http://localhost:$Port"`

2. Create `scripts/stop_windows.ps1`:
- `$ErrorActionPreference = "Stop"`
- Define: `$ContainerName = "finally-app"`
- Stop: `docker stop $ContainerName 2>$null` (ignore errors via try/catch or redirect)
- Remove: `docker rm $ContainerName 2>$null`
- Print: "FinAlly stopped. Data preserved in Docker volume."
- Idempotent (no error if not running)
