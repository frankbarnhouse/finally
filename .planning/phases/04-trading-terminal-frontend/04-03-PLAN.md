Create `src/components/PortfolioHeatmap.tsx` as a "use client" component:

1. **Data source:** Read positions from `usePortfolioStore(s => s.positions)`. Read live prices from `usePriceStore` to get current prices for each position (the API positions have current_price but it may be stale — merge with live SSE prices for real-time updates).

2. **Treemap computation** using d3-hierarchy:
   ```typescript
   import { hierarchy, treemap, treemapSquarify } from 'd3-hierarchy';
   ```
   - Create hierarchy from positions: `hierarchy({ children: positions }).sum(d => d.quantity * d.current_price)`
   - Sort by value descending
   - Apply `treemap().size([width, height]).padding(2).tile(treemapSquarify)`
   - Get leaves for rendering

3. **Color coding:** Interpolate color based on unrealized P&L percentage:
   - Positive P&L: shades of green (#27ae60), intensity proportional to % gain
   - Negative P&L: shades of red (#e74c3c), intensity proportional to % loss
   - Zero/flat: neutral gray (#8b949e)
   - Cap intensity at +/-10% so the colors are always readable

4. **SVG rendering:** Render as an SVG element:
   - Each leaf becomes a `<rect>` with x, y, width, height from the treemap layout
   - Each rect has a `<text>` child showing the ticker symbol (centered, truncated if rect too small)
   - Small rects (width < 40 or height < 25) skip the text label
   - Show P&L value below ticker if space permits

5. **Sizing:** Use a ref + ResizeObserver to get container dimensions. Recompute treemap on resize.

6. **Empty state:** When no positions exist, show muted text: "No positions yet"

7. **Styling:** Component header "PORTFOLIO" in muted text. Container has bg-terminal-surface, border-terminal-border.
